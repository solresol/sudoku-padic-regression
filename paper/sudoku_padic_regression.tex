\begin{abstract}
This note studies a general $p$-adic encoding of finite-domain all-different systems and then uses Sudoku as a concrete case study. The gain of the encoding is representational rather than algorithmic: it keeps optimisation in the native $81$-cell digit space, avoids a one-hot lift to $729$ Boolean indicators, and makes the objective readable in terms of the puzzle state itself. Given a graph whose vertices carry finite allowed sets and whose edges reward unequal assignments, we show that a signed synthetic $p$-adic regression objective has global minimisers exactly the domain-respecting assignments that maximise the number of unequal edges. When the corresponding list-colouring instance is satisfiable, the global minimisers are precisely the proper list-colourings.
Sudoku fits this template by taking one variable per cell, edges between cells that share a row, column, or box, and singleton domains on clue cells. To make the regression viewpoint explicit, we write the unary penalties as synthetic observations that ``snap'' each coefficient to a digit and use negative regularisation on pairwise differences to reward inequality. For primes $p>9$, the $p$-adic norm makes every nonzero digit difference have norm $1$, eliminating magnitude bias and turning ``difference'' into a clean proxy for ``not equal''. Once the state is digit-valued, the loss is just an additive constant plus the number of peer conflicts.
We then ``train'' the resulting regression instance using two generic optimisers: a stepwise/coordinate-descent heuristic (row-wise swap moves) and a Zubarev-style stochastic walk that samples updates with probability proportional to $\exp(-\beta\,\Delta L)$. These heuristics are not competitive with dedicated Sudoku methods, but they make the encoding's central feature concrete: the state is the puzzle itself, and the effect of a local move can be audited directly in the affected columns and boxes without unpacking an auxiliary Boolean representation.
\end{abstract}

\section{Motivation and prior work}
There are many sensible ways to solve Sudoku puzzles. Exact-cover formulations solved by Knuth's Algorithm X with dancing links \cite{knuth2000dancing}, constraint-programming formulations built around \emph{alldifferent} constraints \cite{simonis2005sudoku}, and SAT encodings \cite{lynce2006sudoku} are all natural approaches. This paper is not one of them.

Instead, it asks whether modifying the loss landscape of linear regression is sufficient to encode finite-domain all-different systems using digit-valued parameters directly rather than one-hot indicators, with Sudoku as the cleanest worked example. The point is not computational efficiency but representational parsimony: use one variable per semantic object, keep the domain of each variable aligned with the problem domain, and let a single $p$-adic objective enforce both domain membership and pairwise distinctness. On digit-valued states the loss becomes a constant plus a visible conflict count, so local updates can be followed in the native Sudoku grid rather than through an auxiliary encoding.

This is the sense in which the construction is intended to be hand-traceable. The claim is not that a human would solve Sudoku by imitating the optimiser; it is that once the state is represented by digits directly, the effect of a proposed local move can be checked by hand from the affected units alone.

Two earlier results motivate the construction:
\begin{itemize}
    \item A structural result on $p$-adic linear regression \cite{baker2025padic}: an optimal hyperplane is supported by $n+1$ data points.
    \item A complexity result on $p$-adic linear regression \cite{baker2026nphard}: the problem is NP-hard, via an embedding of \textsc{Max-Cut} using positive regularisation points that snap coefficients to $\{0,1\}$.
\end{itemize}
Our contribution in this paper is the use of negative regularisation to enforce pairwise distinctness, together with a Sudoku case study that makes the native-state and locally auditable character of the encoding concrete. The construction in Section~\ref{construction} isolates the general mechanism behind the Sudoku example.

\section{Preliminaries}

\subsection{$p$-adic integers and norms}
Fix a prime $p$. The $p$-adic valuation $v_p(n)$ of an integer $n\neq 0$ is the exponent of $p$ in its factorisation, and the $p$-adic absolute value is
\[
|n|_p = p^{-v_p(n)}, \qquad |0|_p = 0.
\]
We will work in the $p$-adic integers $\mathbb{Z}_p$, where all norms satisfy $|x|_p\le 1$. The key choice for Sudoku is to take $p>9$ (e.g.\ $p=11$), so that the $p$-adic absolute value of any Sudoku digit or digit difference is always \(0\) or \(1\).

\section{A general all-different regression template}
\label{construction}
\subsection{List colouring}
Many finite-domain constraint systems built from all-different conditions can be viewed as list-colouring problems on a graph. Each variable $x_i$ is assigned a value from a finite allowed set $D_i$, and an edge $\{i,j\}$ expresses the preference or requirement that $x_i\neq x_j$.

Figure~\ref{fig:list-colouring-example} gives a concrete toy instance. Its synthetic dataset has six positive observations, one for each allowed value in the three domains, and three negative observations, one for each edge. The proper list-colouring \(x_1=1\), \(x_2=2\), \(x_3=3\) therefore activates every edge reward while incurring only the unavoidable unary baseline.

\subsection{Regression formulation}
To make the regression viewpoint explicit, it is useful to describe the \emph{synthetic dataset} rather than only the resulting loss. In the present construction there is no ambient collection of ordinary feature-response pairs. Every observation is inserted deliberately, either to pin a variable to its allowed values or to reward separation along an edge.

For the toy instance in Figure~\ref{fig:list-colouring-example}, this equivalence can be written out explicitly. Figure~\ref{fig:list-colouring-regression} shows the corresponding synthetic dataset: six positive observations encode the three domains, and three negative observations encode the three edges.

Over $\mathbb{Z}_p$ with $p>q$, the $p$-adic norm is perfectly suited to this setting: once values are restricted to $\{1,\dots,q\}$, every nonzero difference has norm $1$.

\begin{definition}[Signed synthetic $p$-adic regression dataset]
\label{def:signed-dataset}
For parameters \(x\in \mathbb Z_p^n\), a signed weighted affine observation is a quadruple \((\sigma,u,b,\gamma)\) with \(\sigma\in\{+1,-1\}\), \(u\in\mathbb Z^n\), \(b\in\mathbb Z\), and \(\gamma>0\). It contributes the term
\[
\sigma\gamma\,|u^\top x-b|_p
\]
to the loss; formally, this is the $p$-adic analogue of a weighted Euclidean residual term.
\end{definition}

\begin{definition}[Positive/negative observations]
A signed weighted affine observation \((\sigma,u,b,\gamma)\) is \emph{positive} if \(\sigma=+1\) and \emph{negative} if \(\sigma=-1\). We also use the phrases \emph{positive regularisation term} and \emph{negative regularisation term}.
\end{definition}

\begin{definition}[Synthetic dataset for list-colouring]
\label{def:list-dataset}
Let \(V=\{1,\dots,n\}\), let \(G=(V,E)\) be a finite graph, let \(D_i\subseteq \{1,\dots,q\}\) be nonempty allowed sets, and fix a prime \(p>q\). Define
\[
w_{ii}:=0,
\qquad
w_{ij}:=
\begin{cases}
1, & \{i,j\}\in E,\\
0, & \{i,j\}\notin E,
\end{cases}
\qquad (i\neq j),
\]
let \(\Delta:=\max_{i\in V}\sum_{j=1}^n w_{ij}\), and set \(\lambda:=1+\Delta\). Writing \(e_1,\dots,e_n\) for the standard basis vectors of \(\mathbb Z^n\), define
\[
\mathcal S^+ := \{(+1,e_i,a,\lambda): i\in V,\ a\in D_i\},
\]
\[
\mathcal S^- := \{(-1,e_i-e_j,0,1): 1\le i<j\le n,\ w_{ij}=1\}.
\]
Set \(\mathcal S:=\mathcal S^+\sqcup\mathcal S^-\).
\end{definition}

\begin{definition}[Loss associated with the list-colouring dataset]
\label{def:list-loss}
For \(\mathcal S\) as in Definition~\ref{def:list-dataset}, define
\[
L_{\mathcal S}(x)
=
\lambda\sum_{i=1}^n \sum_{a\in D_i}|x_i-a|_p
-\sum_{1\le i<j\le n} w_{ij}|x_i-x_j|_p.
\]
\end{definition}

We now analyse that construction. This dataset has exactly \(\sum_{i=1}^n |D_i|\) positive observations and \(|E|\) negative observations, so it is computable in time \(O(\sum_i |D_i|+|E|)\). The positive observations pin coordinates to allowed values, the negative observations reward inequality across edges, and there are no ordinary data points unrelated to the constraint system.

Each positive observation \((+1,e_i,a,\lambda)\) contributes \(\lambda|x_i-a|_p\) to the loss, and each negative observation \((-1,e_i-e_j,0,1)\) contributes \(-|x_i-x_j|_p\) to the loss. Summing them yields the displayed loss. The size bound and runtime are immediate from the explicit definitions.

\subsection{Correctness}
Since \(p>q\), the residues of \(1,\dots,q\) are distinct modulo \(p\). In particular, for any fixed \(i\in V\) and any \(t\in\mathbb Z_p\), there is at most one \(a\in D_i\) with \(t\equiv a\pmod p\).

First we record the simple observation that, on \(\{1,\dots,q\}\), the \(p\)-adic norm detects inequality exactly.

\begin{lemma}[Edge terms become inequality indicators on the domain]
\label{lem:edge-indicator}
If \(x_i,x_j\in \{1,\dots,q\}\) and \(p>q\), then
\[
|x_i-x_j|_p=
\begin{cases}
0,&x_i=x_j,\\
1,&x_i\neq x_j.
\end{cases}
\]
\end{lemma}

\begin{proof}
If \(x_i=x_j\) then the difference is zero. If \(x_i\neq x_j\), then \(1\le |x_i-x_j|\le q-1<p\), so \(p\nmid (x_i-x_j)\) and therefore \(|x_i-x_j|_p=1\).
\end{proof}

We next prove a digit-snapping lemma: if \(x\) minimises \(L_{\mathcal S}\), then each coordinate \(x_i\) lies in \(D_i\). The explicit unary-well formula makes that argument transparent.

\begin{lemma}[Unary wells]
\label{lem:unary-well}
For \(i\in V\), define
\[
U_i(t):=\sum_{a\in D_i}|t-a|_p
\qquad (t\in\mathbb Z_p).
\]
Then \(U_i(t)\ge |D_i|-1\), with equality if and only if \(t\in D_i\).
\end{lemma}

\begin{proof}
If \(t\in D_i\), then one summand in \(U_i(t)\) is \(0\), and the remaining \(|D_i|-1\) summands are \(1\), so \(U_i(t)=|D_i|-1\).

If \(t\notin D_i\) but \(t\equiv b \pmod p\) for some \(b\in D_i\), then \(0<|t-b|_p<1\), while for every \(a\in D_i\setminus\{b\}\) we have \(t-a\equiv b-a\not\equiv 0\pmod p\), so \(|t-a|_p=1\). Hence
\[
U_i(t)=(|D_i|-1)+|t-b|_p > |D_i|-1.
\]

If \(t\) is congruent modulo \(p\) to no element of \(D_i\), then \(|t-a|_p=1\) for all \(a\in D_i\), so \(U_i(t)=|D_i|>|D_i|-1\).
\end{proof}

\begin{lemma}[Digit-snapping lemma]
\label{lem:digit-snapping}
Let \(x\in\mathbb Z_p^n\). Suppose there exists \(i\in V\) with \(x_i\notin D_i\). Choose \(b\in D_i\) as follows: if \(x_i\) is congruent modulo \(p\) to an element of \(D_i\), take that unique element; otherwise choose any \(b\in D_i\). Let \(x'\) be obtained from \(x\) by replacing \(x_i\) with \(b\). Then
\[
L_{\mathcal S}(x')<L_{\mathcal S}(x).
\]
\end{lemma}

\begin{proof}
The positive part of the loss changes by
\[
\lambda\left(\sum_{a\in D_i}|b-a|_p-\sum_{a\in D_i}|x_i-a|_p\right).
\]
There are two cases to consider.

If \(x_i\) is congruent modulo \(p\) to no element of \(D_i\), then \(|x_i-a|_p=1\) for every \(a\in D_i\), so \(\sum_{a\in D_i}|x_i-a|_p=|D_i|\). On the other hand, one summand in \(\sum_{a\in D_i}|b-a|_p\) is \(0\) and the remaining \(|D_i|-1\) summands are \(1\), so \(\sum_{a\in D_i}|b-a|_p=|D_i|-1\). Hence the positive change is \(-\lambda\). Since \(|x_i-b|_p=1\) in this case, this is at most \(-\lambda |x_i-b|_p\).

If \(x_i\equiv b \pmod p\) for some \(b\in D_i\), then \(\sum_{a\in D_i}|b-a|_p=|D_i|-1\), whereas
\[
\sum_{a\in D_i}|x_i-a|_p=(|D_i|-1)+|x_i-b|_p.
\]
Hence the positive change is \(-\lambda |x_i-b|_p\).

For each \(j\in V\), the corresponding negative component changes by
\[
w_{ij}\bigl(|x_i-x_j|_p-|b-x_j|_p\bigr).
\]
By the ultrametric inequality,
\[
|x_i-x_j|_p \le \max\bigl(|x_i-b|_p,\ |b-x_j|_p\bigr),
\]
so
\[
|x_i-x_j|_p-|b-x_j|_p \le |x_i-b|_p.
\]
Summing over all \(j\in V\),
\[
L_{\mathcal S}(x')-L_{\mathcal S}(x)
\le
\left(-\lambda+\sum_{j=1}^n w_{ij}\right)|x_i-b|_p
\le
-|x_i-b|_p
<
0.
\]
\end{proof}

Next we analyse the loss on domain-respecting assignments. In that regime it becomes a constant plus a conflict count, so if the list-colouring instance is satisfiable the rightmost sum can be zero.

\begin{theorem}[Correctness of the synthetic reduction]
\label{thm:all-different}
For the dataset \(\mathcal S=\mathcal S^+\sqcup\mathcal S^-\) from Definition~\ref{def:list-dataset} and the loss from Definition~\ref{def:list-loss}, every global minimiser of \(L_{\mathcal S}\) lies in \(\prod_{i=1}^n D_i\). On \(\prod_{i=1}^n D_i\),
\[
L_{\mathcal S}(x)
=
\lambda\sum_{i=1}^n (|D_i|-1)
-|E|
+\sum_{1\le i<j\le n} w_{ij}\mathbf 1_{x_i=x_j}.
\]
Consequently, the global minimisers are exactly the domain-respecting assignments that minimise the conflict sum \(\sum_{1\le i<j\le n} w_{ij}\mathbf 1_{x_i=x_j}\), or equivalently maximise the number of unequal edges. In particular, if the list-colouring instance \((G,(D_i)_{i=1}^n)\) is satisfiable, then this minimum conflict sum is \(0\), and the global minimisers are exactly its proper list-colourings.
\end{theorem}

\begin{proof}
By Lemma~\ref{lem:digit-snapping}, any point with some coordinate outside its domain can be improved by snapping that coordinate into the domain. Therefore every global minimiser lies in \(\prod_i D_i\). If \(x\in \prod_i D_i\), then Lemma~\ref{lem:unary-well} gives \(U_i(x_i)=|D_i|-1\) for every \(i\), and Lemma~\ref{lem:edge-indicator} gives
\[
|x_i-x_j|_p=\mathbf 1_{x_i\neq x_j}
\qquad (1\le i<j\le n).
\]
Since \(\mathbf 1_{x_i\neq x_j}=1-\mathbf 1_{x_i=x_j}\) on every edge, substituting these identities into \(L_{\mathcal S}\) yields the displayed formula. The final claims follow immediately.
\end{proof}

\begin{corollary}
\label{cor:all-different-csp}
Any finite-domain all-different constraint system can be encoded, in polynomial time, as the global minimiser set of a signed synthetic $p$-adic regression dataset.
\end{corollary}

\begin{proof}
Given an all-different constraint system, form its primal graph: the vertices are the variables, and two variables are adjacent if they appear together in some all-different constraint. The satisfying assignments are exactly the proper list-colourings of this graph, with the variable domains as lists. Apply Definitions~\ref{def:list-dataset} and \ref{def:list-loss}, and Theorem~\ref{thm:all-different}.
\end{proof}

\section{Sudoku as a special case}
\subsection{Sudoku}
A (standard) Sudoku puzzle consists of a partially filled $9\times 9$ grid. The task is to assign a digit in $\{1,\dots,9\}$ to each empty cell such that:
\begin{enumerate}
    \item each row contains all digits $1,\dots,9$ exactly once;
    \item each column contains all digits $1,\dots,9$ exactly once; and
    \item each $3\times 3$ box contains all digits $1,\dots,9$ exactly once.
\end{enumerate}
Equivalently, in each unit (row, column, box) all entries are \emph{pairwise distinct} and each entry lies in $\{1,\dots,9\}$.

Standard Sudoku fits Corollary~\ref{cor:all-different-csp} directly. The vertices are the $81$ cells, two vertices are adjacent when the corresponding cells share a row, column, or $3\times 3$ box, unclued cells have domain $\{1,\dots,9\}$, and clue cells have singleton domains. A proper list-colouring of this graph is exactly a valid Sudoku completion.

The general lemmas above specialise very cleanly to Sudoku. We therefore state the Sudoku facts below mainly as direct corollaries of the unary-well and edge-indicator lemmas, and only then write down the concrete synthetic datasets used in the paper.

\subsection{Positive regularisation: snapping coefficients to digits}

Let $D=\{1,2,\dots,9\}\subset \mathbb{Z}_p$, and write the Sudoku variables as a vector
\[
x \in \mathbb{Z}_p^{81},
\]
with one coefficient per cell.

\begin{definition}[Sudoku digit-snapping penalty]
For $p>9$, define the digit-snapping penalty
\[
U_D(t) := \sum_{k=1}^9 |t-k|_p.
\]
\end{definition}

By Lemma~\ref{lem:unary-well} applied to the common domain \(D\), one has \(U_D(t)\ge 8\) for every \(t\in\mathbb Z_p\), with equality if and only if \(t\in D\).

This is the $9$-valued analogue of the earlier ``snap to $\{0,1\}$'' construction from \cite{baker2026nphard}: the synthetic observations $t=k$ for all $k\in D$ create a multiwell objective with minima exactly at $D$.

\subsection{Negative regularisation: rewarding inequality via $p$-adic norms}

The remaining Sudoku constraints are that entries in each row/column/box are pairwise unequal. Pairwise inequality can be encoded using differences $x_i-x_j$.

\begin{corollary}[Sudoku differences as inequality indicators]
\label{cor:sudoku-differences}
Let $p>9$ and $a,b\in D$. Then
\[
|a-b|_p =
\begin{cases}
0 & a=b,\\
1 & a\neq b.
\end{cases}
\]
\end{corollary}

\begin{proof}
This is Lemma~\ref{lem:edge-indicator} with \(q=9\).
\end{proof}

\subsection{Minimal and expanded regression objectives}
Let $\mathcal{P}$ be the set of cell-pairs $(i,j)$ that share a unit (same row, same column, or same box). Let $\mathcal{C}$ be the set of given clues; for $i\in\mathcal{C}$ let $g_i\in D$ be the fixed digit.

Specialising Definitions~\ref{def:list-dataset} and \ref{def:list-loss}, together with Theorem~\ref{thm:all-different}, to the Sudoku graph gives the minimal synthetic dataset
\begin{align}
L_{\min}(x) &=
\alpha\sum_{i\notin\mathcal{C}}\sum_{k=1}^9 |x_i-k|_p
\;+\;
\alpha\sum_{i\in\mathcal{C}} |x_i-g_i|_p
\;-\;
\sum_{(i,j)\in\mathcal{P}} |x_i-x_j|_p.
\label{eq:objective-minimal}
\end{align}
A separate coefficient on the pairwise term is unnecessary here: any positive edge weight can be scaled out, so we normalise that coefficient to \(1\). Thus \(\alpha\) itself plays the role of the general pinning weight \(\lambda\), and any choice \(\alpha>20\) satisfies the domination condition from Theorem~\ref{thm:all-different}. Every term in \eqref{eq:objective-minimal} is a regularisation term with a specific purpose: the positive terms pin cells to allowed digits or clues, and the negative terms reward unequal peer values. There are no ordinary feature-response observations beyond these synthetic regularisers. The minimal dataset therefore contains \(9(81-|\mathcal C|)+|\mathcal C|=729-8|\mathcal C|\) positive observations and \(810\) negative observations. This is the conceptual formulation used in the correctness statements and in the parsimony discussion below: a single scalar objective on the native digit state space.

For the experiments, however, we use the slightly more redundant expanded objective
\begin{align}
L(x) &=
\alpha\sum_{i=1}^{81}\sum_{k=1}^9 |x_i - k|_p
\;+\;
\alpha_{\mathrm{clue}}\sum_{i\in\mathcal{C}} |x_i - g_i|_p
\;-\;
\sum_{(i,j)\in\mathcal{P}} |x_i-x_j|_p.
\label{eq:objective}
\end{align}
Equation \eqref{eq:objective} keeps the nine digit-well residuals visible even on clue cells and then adds the clue pins separately. It therefore contains \(729+|\mathcal C|\) positive observations and the same \(810\) negative observations. The first two sums are \emph{positive} regularisation, and the final sum is \emph{negative} regularisation. There is still no ``ordinary'' data in the usual regression sense: every synthetic observation exists only to enforce a Sudoku constraint. Equivalently, counting peer pairs per unit gives $27\binom{9}{2}=972$ negative terms, but $162$ of these pairs lie simultaneously in a row/column \emph{and} a box, leaving $810$ distinct pairs overall. We use this expanded form only as an implementation convenience; it should be read as a redundant refinement of \eqref{eq:objective-minimal}, not as the more conceptually fundamental model.

Once \(x\) is digit-valued, the pairwise part is simply
\[
-\sum_{(i,j)\in\mathcal{P}} |x_i-x_j|_p
=
-|\mathcal P|+\sum_{(i,j)\in\mathcal{P}}\mathbf 1_{x_i=x_j},
\]
so the loss differs from the peer-conflict count only by an additive constant.

The $810$ pairwise terms are essentially irreducible if one insists on \emph{pairwise} penalties: omitting a single peer-pair $(i,j)$ removes the only residual that directly distinguishes $x_i=x_j$ from $x_i\neq x_j$ once $x$ is digit-valued, and no linear ``hash'' of the remaining differences can substitute because nonvanishing is not preserved under linear combinations (e.g.\ $a-b\neq 0$ and $b-c\neq 0$ does not prevent $a-c=0$).

\begin{corollary}[Sudoku completion as a special case]
\label{cor:sudoku-special-case}
Let \(G_{\mathrm{Sud}}\) be the graph on the \(81\) cells of a Sudoku puzzle, with an edge between two cells exactly when they share a row, column, or \(3\times 3\) box. For a puzzle with clue set \(\mathcal{C}\) and clue digits \(g_i\), define
\[
D_i :=
\begin{cases}
\{g_i\}, & i\in \mathcal{C},\\
\{1,\dots,9\}, & i\notin \mathcal{C}.
\end{cases}
\]
Assume \(\alpha>20\). Then the global minimisers of the minimal objective \eqref{eq:objective-minimal} are exactly the valid completions of the Sudoku puzzle.
\end{corollary}

\begin{proof}
Each row, column, and box of Sudoku forms a clique of size \(9\) in \(G_{\mathrm{Sud}}\). Thus a proper list-colouring assigns pairwise distinct digits in each unit. Since the available values are exactly \(\{1,\dots,9\}\), pairwise distinctness in a unit is equivalent to containing every digit exactly once. The singleton domains enforce the clues. Apply Theorem~\ref{thm:all-different}.
\end{proof}

For Sudoku, this bookkeeping is completely explicit. In the deduplicated peer graph, every cell has degree \(20\), so Theorem~\ref{thm:all-different} requires \(\alpha>20\) for the minimal objective \eqref{eq:objective-minimal}; a convenient sufficient choice is \(\alpha=21\). The same sufficient condition \(\alpha>20\) works for the expanded objective \eqref{eq:objective}, because its extra clue terms only strengthen the positive regularisation. If instead one rewrote the loss as a sum over row, column, and box scopes and counted overlaps separately, each cell would contribute \(8+8+8=24\) incident negative terms, and the corresponding sufficient choice would be \(\alpha>24\). This is the precise place where the negative regularisation has to be dominated by the positive unary weight.

\paragraph{Why $p$-adics help with negative weights.}
If one tries to do this with an ordinary squared-loss term $-(x_i-x_j)^2$ in $\mathbb{R}$, the optimiser is perversely incentivised to make some differences \emph{large}, not merely nonzero, and the objective can become unstable unless heavily bounded. With $p>9$ and digit-valued variables, $|x_i-x_j|_p\in\{0,1\}$, removing magnitude bias.

\section{A stepwise/coordinate-descent heuristic}

The objective \eqref{eq:objective} induces an NP-hard global optimisation problem (consistent with the earlier NP-hardness result for $p$-adic regression). The goal here is not a worst-case polynomial-time algorithm, but to test whether generic regression-style updates can still solve individual puzzles.

Assume now that \(\alpha>20\). On digit-valued assignments consistent with the clues, the two positive sums in \eqref{eq:objective} are constant, so minimising \(L\) is equivalent to minimising the remaining column/box conflict count. In implementation we therefore restrict search to digit-valued assignments consistent with clues. Concretely, we initialise each row as a random permutation of $1,\dots,9$ consistent with its clues, and then apply row-wise swaps that reduce the remaining loss (column/box conflicts). This is a form of coordinate descent on an integer lattice.

We will call this move set \emph{hand-traceable}: the state is a filled Sudoku grid, and the effect of a candidate swap can be checked by inspecting only the units that the swap touches, rather than by decoding an auxiliary indicator representation.

\begin{lemma}[Locality of row swaps]
\label{lem:row-swap-locality}
Let \(x\) be a digit-valued, clue-consistent Sudoku state in which each row is a permutation of \(1,\dots,9\). Let \(\xi\) swap two non-clue entries in a single row \(r\), at columns \(c_1\neq c_2\). Then the change
\[
\Delta L(\xi):=L(x\oplus \xi)-L(x)
\]
is determined entirely by the conflict-pair contributions from columns \(c_1\) and \(c_2\), together with the one or two boxes containing \((r,c_1)\) and \((r,c_2)\). No other term of the digit-restricted objective changes.
\end{lemma}

\begin{proof}
On digit-valued clue-consistent states, the positive part of \eqref{eq:objective} is constant, so only the column/box conflict count matters. A within-row swap preserves the row permutation, so row conflicts remain zero. Every cell outside row \(r\) is unchanged, and within row \(r\) only the two swapped positions change value. Hence only the two corresponding columns can change their conflict counts. Likewise, only the boxes containing those two cells can change. All other columns and boxes see the same multiset of digits before and after the swap, so their contributions are unchanged.
\end{proof}

\begin{algorithm}[t]
\caption{Row-swap stepwise regression (heuristic)}
\begin{algorithmic}[1]
\Require Puzzle with clues $\mathcal{C}$, max steps $T$, restarts $R$
\For{$r=1$ to $R$}
    \State Initialise each row as a permutation of $1,\dots,9$ consistent with its clues
    \For{$t=1$ to $T$}
        \If{no column/box conflicts} \State \Return solution \EndIf
        \State Choose a row involved in a conflict
        \State Find the best swap of two non-clue cells in that row (largest loss decrease)
        \State Apply the swap (or a random swap if no improving swap exists)
    \EndFor
\EndFor
\State \Return best assignment found
\end{algorithmic}
\end{algorithm}

This ``stepwise regression'' view matches the informal idea: at each step, choose the coefficient change that gives the largest incremental loss improvement. Lemma~\ref{lem:row-swap-locality} is the precise bridge from the $p$-adic formulation to hand-traceability: in the row-permutation regime, a move is scored by a small local audit of the affected columns and boxes.

\section{A Zubarev-style stochastic walk}
The ``stepwise regression'' heuristic above is tailored to the strong-snapping Sudoku setting: evaluate all within-row swaps and greedily choose the best improving move (or a random move when stuck). Once we take seriously the idea that Sudoku is being treated as a regression instance, it is natural---if delightfully over the top---to also try a generic training method from the $p$-adic regression literature.

\subsection{Zubarev's proposal in the regression setting}
In \cite{zubarev2025padic}, Zubarev notes that loss functions built from sums of $p$-adic norms are locally constant almost everywhere, so gradient methods are uninformative. He therefore suggests a stochastic alternative: ``we can propose a stochastic optimization algorithm for the function (9) based on a random walk in the parameter space $w$''~\cite{zubarev2025padic}.

Concretely, the proposed training dynamics is a discrete-time random process
\begin{equation}
w_{t+1} = w_t + \xi_t(w_t,\beta_t),
\label{eq:zubarev-walk}
\end{equation}
where $\beta_t>0$ is a ``continuity'' (inverse temperature) parameter and $\xi_t$ is sampled from a distribution biased toward loss-decreasing moves. In Zubarev's formulation the conditional law has density (with respect to the $p$-adic Haar measure)
\begin{equation}
P(\xi \mid w,\beta)\ \propto\ \exp\!\bigl(-\beta\,(L(w+\xi)-L(w))\bigr).
\label{eq:zubarev-boltzmann}
\end{equation}
Under mild conditions (in particular, that the set of improving moves has nonzero measure), Zubarev shows there is a regime of $\beta$ values for which the expected loss decreases, i.e.\ $L(w_t)$ is a strict supermartingale~\cite{zubarev2025padic}.

\subsection{A discrete Sudoku analogue}
In the digit-restricted implementation used here, the state is already digit-valued and clue-consistent. We therefore treat the Zubarev walk as a \emph{finite-move} stochastic optimiser:
\begin{itemize}
    \item The parameter vector $w$ is the current Sudoku filling $x\in D^{81}$.
    \item The loss $L(x)$ is the number of conflicting pairs in columns and boxes (equivalently, \eqref{eq:objective} up to constants in the digit-restricted regime).
    \item A move $\xi$ is a swap of two non-clue cells within a single row (preserving each row as a permutation of $1,\dots,9$).
\end{itemize}

For a chosen row $r$ and current state $x$, let $\Xi_r(x)$ be the set of all such swaps, and let $\Delta L(\xi) := L(x\oplus\xi)-L(x)$ be the change in loss after applying swap $\xi$ (where $x\oplus\xi$ denotes ``apply the swap''). Replacing the Haar measure in \eqref{eq:zubarev-boltzmann} with the counting measure on the finite set $\Xi_r(x)$ gives the discrete sampling rule
\begin{equation}
\Pr(\xi \mid x,\beta_t) = \frac{\exp\!\bigl(-\beta_t\,\Delta L(\xi)\bigr)}{\sum_{\xi'\in\Xi_r(x)} \exp\!\bigl(-\beta_t\,\Delta L(\xi')\bigr)}.
\label{eq:zubarev-discrete}
\end{equation}
When $\beta_t\to 0$ this is uniform random swapping; when $\beta_t\to\infty$ it concentrates on the best (most loss-decreasing) swap, recovering a soft version of the greedy stepwise algorithm. In practice we use a simple annealing schedule: $\beta_t$ increases linearly from $\beta_0$ to $\beta_1$ over the allotted steps, gradually shifting from exploration to exploitation.

\begin{algorithm}[t]
\caption{Zubarev walk over row swaps (heuristic)}
\begin{algorithmic}[1]
\Require Puzzle with clues $\mathcal{C}$, max steps $T$, restarts $R$, schedule $\{\beta_t\}_{t=1}^T$
\For{$r=1$ to $R$}
    \State Initialise each row as a permutation of $1,\dots,9$ consistent with its clues
    \For{$t=1$ to $T$}
        \If{no column/box conflicts} \State \Return solution \EndIf
        \State Choose a row involved in a conflict
        \State Enumerate row swaps $\Xi_r(x)$ and compute $\Delta L(\xi)$ for each $\xi\in\Xi_r(x)$
        \State Sample $\xi\in\Xi_r(x)$ with probability $\propto \exp(-\beta_t\,\Delta L(\xi))$
        \State Apply swap $\xi$
    \EndFor
\EndFor
\State \Return best assignment found
\end{algorithmic}
\end{algorithm}

\section{Parsimony relative to SAT/CSP encodings}

A standard SAT encoding introduces Boolean variables
\[
y_{r,c,d}\in\{0,1\},
\]
with the intended meaning that cell $(r,c)$ contains digit $d$. This gives
\[
9\times 9\times 9 = 729
\]
variables before any clues are applied.

In a direct CNF model, four families of exactly-one constraints are required:
\begin{enumerate}
    \item one digit per cell ($81$ constraints);
    \item one occurrence of each digit per row ($81$ constraints);
    \item one occurrence of each digit per column ($81$ constraints);
    \item one occurrence of each digit per box ($81$ constraints).
\end{enumerate}
Flattening each exactly-one constraint on $9$ literals into CNF gives one $9$-literal clause plus $\binom{9}{2}=36$ binary exclusion clauses. Hence the expanded SAT input contains
\[
324 \text{ long clauses} \;+\; 324\cdot 36 = 11{,}664 \text{ binary clauses},
\]
for a total of
\[
11{,}988 + |\mathcal{C}|
\]
clauses after adding one unit clause for each clue in $\mathcal{C}$.

There are three distinct parsimony claims here, and it helps to keep them separate.

\paragraph{Parameter parsimony.}
The exact-cover / SAT lift uses $729$ indicator variables, whereas the $p$-adic model keeps the puzzle in its native $81$-cell digit space. This is the cleanest comparison against lifted encodings: one digit-valued variable per semantic object rather than nine Boolean indicators per cell.

\paragraph{Objective form.}
For Sudoku, the minimal objective \eqref{eq:objective-minimal} is already a single scalar loss on that native state space. The expanded implementation \eqref{eq:objective} contains
\[
729 \text{ digit-snapping residuals} \;+\; |\mathcal{C}| \text{ clue residuals} \;+\; 810 \text{ pairwise residuals},
\]
so the fully expanded regression instance has
\[
1{,}539 + |\mathcal{C}|
\]
scalar terms. Relative to naive CNF, this is a factor of about
\[
\frac{11{,}988}{1{,}539} \approx 7.8
\]
fewer expanded constraints, together with a factor of
\[
\frac{729}{81} = 9
\]
fewer variables. Relative to exact cover, the comparison is subtler: the regression model still avoids the $729$-variable lift, but it has more scalar terms than the $324$ structured constraints.

\paragraph{Cognitive parsimony.}
High-level constraint-programming models deserve a separate remark. A structured CP model based on \(81\) cell variables and \emph{alldifferent} constraints is already semantically close to the puzzle, so the present claim is not that the $p$-adic encoding uses fewer variables than every CP formulation. Rather, its advantage is that it packages the problem as a single native-state objective whose digit-restricted form is just a constant plus peer conflicts. In the row-permutation regime, the current search state is literally a Sudoku grid, and a proposed move can be audited by inspecting only the affected columns and boxes, as in Lemma~\ref{lem:row-swap-locality}. That is a claim about representational readability, not a theorem about runtime.

\section{Discussion and extensions}
The model is still intentionally ``the hard way'': it embeds Sudoku into a (potentially) NP-hard regression-like optimisation problem rather than exploiting Sudoku structure directly. But the point of doing so is representational rather than comic. The interesting feature is that the encoding stays in the puzzle's own digit space while preserving a locally interpretable objective.

\paragraph{Training Sudoku as a regression model.}
The most striking aspect of the construction is arguably not the loss itself, but the fact that once we write down \eqref{eq:objective}, we can treat Sudoku completion as a \emph{training problem} and import generic training dynamics from the $p$-adic regression setting. The Zubarev walk is a particularly clean example: a loss-biased random walk that is not designed for Sudoku in any way, yet still finds valid completions when run over the row-swap move set. What it preserves from the underlying encoding is not Sudoku expertise, but the native-state viewpoint: the walk moves directly in the space of digit fillings rather than in an auxiliary one-hot lift.

\subsection{An archived sparse alternative: powers-of-two sums}
The objective \eqref{eq:objective} uses a large collection of pairwise inequality terms: $|\mathcal{P}|=810$ distinct peer pairs (equivalently $27\binom{9}{2}=972$ pairs if counted per unit, with overlap). An appealing alternative is to enforce all-different constraints \emph{without} pairwise penalties, by changing the digit representation. This gives a useful contrast case: it is structurally sparser in one sense, but much less local and much less hand-traceable.

Map digits $d\in\{1,\dots,9\}$ to powers of two,
\[
d \longmapsto 2^{d-1} \in A := \{1,2,4,\dots,256\}.
\]
If every cell variable is restricted to $A$, then the condition ``a unit contains each digit exactly once'' is equivalent to a \emph{single} sum constraint:
\[
\sum_{(r,c)\in g} \beta_{r,c} = 1+2+\cdots+256 = 2^9-1 = 511,
\]
for each row/column/box $g$. This collapses the Sudoku constraints to $27$ residuals of the form
\[
r_g(\beta) := 511 - \sum_{(r,c)\in g}\beta_{r,c}.
\]
For a prime $p$, one can then study the extremely parameter-sparse $p$-adic objective
\[
L_p(\beta) := \sum_g |r_g(\beta)|_p,
\]
optionally together with a (soft or hard) regulariser that encourages $\beta_{r,c}\in A$ and enforces the clues.

This encoding also suggests a ``Hensel-ish'' lifting perspective: a constraint is satisfied at lift level $k$ when $r_g(\beta)\equiv 0\pmod{p^k}$ (equivalently $v_p(r_g(\beta))\ge k$), and a heuristic tries to increase the minimum valuation across groups. Once $p^k > 9\cdot 256 = 2304$, having all residuals divisible by $p^k$ forces every residual to be zero, yielding an actual integer solution. In archived experiments with simple local-search moves (greedy single-cell changes and simulated annealing), this objective exhibited abundant local minima and did not reliably solve standard benchmark puzzles. The contrast is instructive: sheer sparsity of structural constraints is not enough. The more successful encoding in the main body of the paper is not merely compact; it is compact while remaining domain-native and locally auditable.

Several extensions are immediate:
\begin{itemize}
    \item \textbf{Confusion-aware digit losses.} If clues came from OCR or handwriting rather than clean symbolic input, one could weaken the loss for commonly confused pairs (for example $1$ versus $7$) instead of treating every wrong digit as equally wrong.
    \item \textbf{Alternative constraint encodings.} Beyond the powers-of-two sum encoding above, one can replace the inequality reward with difference regularisation over the finite set $\{\pm 1,\dots,\pm 8\}$, at the cost of a more complicated objective landscape.
\end{itemize}

\section{Conclusion}
This note takes a deliberately contrarian route to a familiar puzzle: it first shows that finite-domain all-different systems admit tidy $p$-adic regression encodings, and then studies Sudoku as a vivid special case. The construction is mathematically tidy precisely because the $p$-adic norm turns ``different from'' into an indicator-like reward once the variables are digit-valued.

As an optimisation method, this is obviously not competitive with mature Sudoku solvers or SAT/CSP technology. Its interest is representational. The encoding keeps the problem in the native $81$-cell digit space rather than lifting it to $729$ Boolean indicators, yields a single scalar objective whose digit-restricted form is just a constant plus peer conflicts, and supports local move audits such as the row-swap calculations above. Compared with more aggressively compressed alternatives such as the powers-of-two formulation, that locality matters: the successful encoding is not only compact but also readable in the original problem coordinates. In that sense the paper is less about solving Sudoku efficiently than about showing that ultrametric regression can express finite-domain all-different systems in a compact, domain-native, and locally auditable way.
