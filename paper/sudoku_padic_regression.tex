\begin{abstract}
This note studies a general $p$-adic encoding of finite-domain all-different systems and then uses Sudoku as a concrete case study. Given a graph whose vertices carry finite allowed sets and whose edges reward unequal assignments, we show that a $p$-adic regression-style objective has global minimisers exactly the domain-respecting assignments that maximise the total weight of unequal edges. When the corresponding list-colouring instance is satisfiable, the global minimisers are precisely the proper list-colourings.
Sudoku fits this template by taking one variable per cell, edges between cells that share a row, column, or box, and singleton domains on clue cells. To make the regression viewpoint explicit, we expand the unary penalties into synthetic observations that ``snap'' each coefficient to a digit and use negative regularisation on pairwise differences to reward inequality. For primes $p>9$, the $p$-adic norm makes every nonzero digit difference have norm $1$, eliminating magnitude bias and turning ``difference'' into a clean proxy for ``not equal''.
We then ``train'' the resulting regression instance using two generic optimisers: a stepwise/coordinate-descent heuristic (row-wise swap moves) and a Zubarev-style stochastic walk that samples updates with probability proportional to $\exp(-\beta\,\Delta L)$. Despite the layered absurdity---solving Sudoku by first forcing it into a $p$-adic regression mould and then applying generic regression training rules---both heuristics can solve moderate-clue puzzles in small-scale experiments.
\end{abstract}

\section{Motivation and prior work}
There are many sensible ways to solve Sudoku puzzles. Exact-cover formulations solved by Knuth's Algorithm X with dancing links \cite{knuth2000dancing}, constraint-programming formulations built around \emph{alldifferent} constraints \cite{simonis2005sudoku}, and SAT encodings \cite{lynce2006sudoku} are all natural approaches. This paper is not one of them.

Instead, it asks whether modifying the loss landscape of linear regression is sufficient to encode finite-domain all-different systems using digit-valued parameters directly rather than one-hot indicators, with Sudoku as the cleanest worked example. The point is not computational efficiency but conceptual compression: use fewer variables than a typical SAT/CSP encoding, and let the $p$-adic objective enforce both domain membership and pairwise distinctness.

Two earlier results motivate the construction:
\begin{itemize}
    \item A structural result on $p$-adic linear regression \cite{baker2025padic}: an optimal hyperplane is supported by $n+1$ data points.
    \item A complexity result on $p$-adic linear regression \cite{baker2026nphard}: the problem is NP-hard, via an embedding of \textsc{Max-Cut} using regularisation points that snap coefficients to $\{0,1\}$.
\end{itemize}
The central proposition below isolates the general mechanism behind the Sudoku construction.

\section{Preliminaries}

\subsection{Sudoku}
A (standard) Sudoku puzzle consists of a partially filled $9\times 9$ grid. The task is to assign a digit in $\{1,\dots,9\}$ to each empty cell such that:
\begin{enumerate}
    \item each row contains all digits $1,\dots,9$ exactly once;
    \item each column contains all digits $1,\dots,9$ exactly once; and
    \item each $3\times 3$ box contains all digits $1,\dots,9$ exactly once.
\end{enumerate}
Equivalently, in each unit (row, column, box) all entries are \emph{pairwise distinct} and each entry lies in $\{1,\dots,9\}$.

\subsection{$p$-adic integers and norms}
Fix a prime $p$. The $p$-adic valuation $v_p(n)$ of an integer $n\neq 0$ is the exponent of $p$ in its factorisation, and the $p$-adic absolute value is
\[
|n|_p = p^{-v_p(n)}, \qquad |0|_p = 0.
\]
We will work in the $p$-adic integers $\mathbb{Z}_p$, where all norms satisfy $|x|_p\le 1$. The key choice is to take $p>9$ (e.g.\ $p=11$).

\section{A general all-different regression template}
Many finite-domain constraint systems built from all-different conditions can be viewed as list-colouring problems on a graph. Each variable $x_i$ is assigned a value from a finite allowed set $D_i$, and an edge $\{i,j\}$ expresses the preference or requirement that $x_i\neq x_j$. Over $\mathbb{Z}_p$ with $p>q$, the $p$-adic norm is perfectly suited to this setting: once values are restricted to $\{1,\dots,q\}$, every nonzero difference has norm $1$. The next proposition packages that observation into a clean optimisation statement.

\begin{proposition}[Finite-domain all-different systems as $p$-adic regression]
\label{prop:all-different}
Let \(V=\{1,\dots,n\}\), let \(G=(V,E)\) be a finite graph, let \(D_i\subseteq \{1,\dots,q\}\) be nonempty allowed sets, and fix a prime \(p>q\). For \(t\in\mathbb Z_p\), define
\[
R_i(t):=\sum_{a\in D_i}|t-a|_p-(|D_i|-1).
\]
Let \(\lambda_i>0\) for \(i\in V\), and let \(w_{ij}>0\) for \(\{i,j\}\in E\). Consider
\[
L(x):=\sum_{i=1}^n \lambda_i R_i(x_i)-\sum_{\{i,j\}\in E} w_{ij}|x_i-x_j|_p.
\]
Assume that
\[
\lambda_i>\sum_{j:\{i,j\}\in E} w_{ij}
\qquad\text{for all }i.
\]
Then every global minimiser of \(L\) lies in \(\prod_{i=1}^n D_i\). On \(\prod_{i=1}^n D_i\),
\[
L(x)=-\sum_{\{i,j\}\in E} w_{ij}\mathbf 1_{x_i\neq x_j}.
\]
Consequently, the global minimisers are exactly the domain-respecting assignments that maximise the total weight of unequal edges. In particular, if the list-colouring instance \((G,(D_i)_{i=1}^n)\) is satisfiable, then the global minimisers are exactly its proper list-colourings.
\end{proposition}

\begin{proof}
For each \(i\) and \(t\in\mathbb Z_p\), choose \(\pi_i(t)\in D_i\) as follows: if \(t\in D_i\), set \(\pi_i(t)=t\); if \(t\equiv a \pmod p\) for some \(a\in D_i\), choose that \(a\); otherwise choose any element of \(D_i\). Since \(p>q\), the residues of \(1,\dots,q\) are distinct modulo \(p\). Hence
\[
R_i(t)=|t-\pi_i(t)|_p
\qquad\text{for all }i,t.
\]
Indeed, if \(t\in D_i\) then both sides are \(0\); if \(t\equiv a \pmod p\) for some \(a\in D_i\), then the \(a\)-term contributes \(|t-a|_p\) and every other term contributes \(1\); and if \(t\) is congruent modulo \(p\) to no element of \(D_i\), then every term contributes \(1\).

Now let \(x\in\mathbb Z_p^n\), and suppose \(x_i\notin D_i\) for some \(i\). Put \(a:=\pi_i(x_i)\), and let \(x'\) be obtained from \(x\) by replacing \(x_i\) with \(a\). The unary part decreases by exactly
\[
\lambda_i R_i(x_i)=\lambda_i |x_i-a|_p.
\]
For each neighbour \(j\) of \(i\), the corresponding edge term changes by
\[
w_{ij}\bigl(|x_i-x_j|_p-|a-x_j|_p\bigr).
\]
By the ultrametric inequality,
\[
|x_i-x_j|_p \le \max\bigl(|x_i-a|_p,\ |a-x_j|_p\bigr),
\]
so
\[
|x_i-x_j|_p-|a-x_j|_p \le |x_i-a|_p.
\]
Summing over all neighbours of \(i\),
\[
L(x')-L(x)
\le
-\lambda_i |x_i-a|_p
+
\left(\sum_{j:\{i,j\}\in E} w_{ij}\right)|x_i-a|_p.
\]
Equivalently,
\[
L(x')-L(x)
\le
-\left(\lambda_i-\sum_{j:\{i,j\}\in E} w_{ij}\right)|x_i-a|_p.
\]
By hypothesis the right-hand side is strictly negative. Therefore no global minimiser can have \(x_i\notin D_i\), and every global minimiser lies in \(\prod_{i=1}^n D_i\).

If \(x_i\in D_i\subseteq \{1,\dots,q\}\) for all \(i\), then \(R_i(x_i)=0\) for each \(i\). Moreover, for an edge \(\{i,j\}\in E\),
\[
|x_i-x_j|_p=
\begin{cases}
0,&x_i=x_j,\\
1,&x_i\neq x_j,
\end{cases}
\]
because whenever \(x_i\neq x_j\) we have \(1\le |x_i-x_j|\le q-1<p\). This yields
\[
L(x)=-\sum_{\{i,j\}\in E} w_{ij}\mathbf 1_{x_i\neq x_j}.
\]
The final claims follow immediately.
\end{proof}

The proposition separates the mechanism cleanly into two parts. Strong enough unary penalties force each variable onto its prescribed finite domain, and then the pairwise $p$-adic terms collapse to indicator functions for inequality. What remains is exactly a weighted all-different or list-colouring objective.

\begin{corollary}
\label{cor:all-different-csp}
Any finite-domain all-different constraint system can be encoded as the global minimiser set of a $p$-adic regression objective.
\end{corollary}

\begin{proof}
Given an all-different constraint system, form its primal graph: the vertices are the variables, and two variables are adjacent if they appear together in some all-different constraint. The satisfying assignments are exactly the proper list-colourings of this graph, with the variable domains as lists. Apply Proposition~\ref{prop:all-different}.
\end{proof}

\section{Sudoku as a special case}
Standard Sudoku fits Corollary~\ref{cor:all-different-csp} in the most literal possible way. The vertices are the $81$ cells, two vertices are adjacent when the corresponding cells share a row, column, or $3\times 3$ box, unclued cells have domain $\{1,\dots,9\}$, and clue cells have singleton domains. A proper list-colouring of this graph is exactly a valid Sudoku completion.

The objective used in the experiments below is a deliberately expanded version of this general construction. Instead of keeping one compact unary penalty \(R_i\) per variable, it exposes the individual synthetic residuals that build those penalties. This makes the regression interpretation more transparent and is the form most convenient for counting residuals and describing the heuristics.

\subsection{Positive regularisation: snapping coefficients to digits}

Let $D=\{1,2,\dots,9\}\subset \mathbb{Z}_p$, and write the Sudoku variables as a vector
\[
x \in \mathbb{Z}_p^{81},
\]
with one coefficient per cell.

\begin{definition}[Digit-snapping penalty]
For $p>9$, define the digit-snapping penalty
\[
R_p(t) := \sum_{k=1}^9 |t-k|_p.
\]
\end{definition}

\begin{lemma}[Snapping to $D$ for $p>9$]
Assume $p>9$ and $t\in \mathbb{Z}_p$. Then $R_p(t)$ is minimised precisely when $t\in D$. Moreover, $R_p(k)=8$ for $k\in D$ and $R_p(t)>8$ for $t\notin D$.
\end{lemma}

\begin{proof}
If $t=k\in D$, then the $k$th term is $|k-k|_p=0$ and for any $k'\neq k$ we have $1\le |k-k'| \le 8 < p$, so $p\nmid (k-k')$ and hence $|k-k'|_p=1$. Thus $R_p(k)=8$.

Now take $t\notin D$. If $t\not\equiv k \pmod p$ for every $k\in D$, then $p\nmid (t-k)$ for all $k$, so $|t-k|_p=1$ for all $k$ and $R_p(t)=9>8$.
Otherwise, $t\equiv k_0 \pmod p$ for exactly one $k_0\in D$ (distinct digits differ by $<p$ so the residue classes are distinct). Then $|t-k_0|_p=p^{-m}$ for some $m\ge 1$, while for $k\neq k_0$ we still have $|t-k|_p=1$. Hence $R_p(t)=8+p^{-m}>8$.
\end{proof}

This lemma is the $9$-valued analogue of the earlier ``snap to $\{0,1\}$'' construction from \cite{baker2026nphard}: the synthetic observations $t=k$ for all $k\in D$ create a multiwell objective with minima exactly at $D$. In the language of Proposition~\ref{prop:all-different}, it is the uniform-domain case \(D_i=D\).

\subsection{Negative regularisation: rewarding inequality via $p$-adic norms}

The remaining Sudoku constraints are that entries in each row/column/box are pairwise unequal. Pairwise inequality can be encoded using differences $x_i-x_j$.

\begin{lemma}[Differences as inequality indicators for $p>9$]
Let $p>9$ and $a,b\in D$. Then
\[
|a-b|_p =
\begin{cases}
0 & a=b,\\
1 & a\neq b.
\end{cases}
\]
\end{lemma}

\begin{proof}
If $a=b$ the difference is zero. If $a\neq b$, then $1\le |a-b|\le 8<p$, so $p\nmid(a-b)$ and $v_p(a-b)=0$, hence $|a-b|_p=1$.
\end{proof}

\subsection{Expanded regression objective}
Let $\mathcal{P}$ be the set of cell-pairs $(i,j)$ that share a unit (same row, same column, or same box). Let $\mathcal{C}$ be the set of given clues; for $i\in\mathcal{C}$ let $g_i\in D$ be the fixed digit.

We construct a linear regression objective by treating each constraint as a synthetic observation of a linear form $a^\top x$ with target $b$ and $p$-adic residual norm:
\begin{align}
L(x) &=
\alpha\sum_{i=1}^{81}\sum_{k=1}^9 |x_i - k|_p
\;+\;
\alpha_{\mathrm{clue}}\sum_{i\in\mathcal{C}} |x_i - g_i|_p
\;-\;
\beta\sum_{(i,j)\in\mathcal{P}} |x_i-x_j|_p.
\label{eq:objective}
\end{align}
Equation \eqref{eq:objective} can be read as a regression problem with an explicit set of synthetic observations: $81$ multiwell ``snap to digit'' terms (implemented as $81\cdot 9=729$ residuals $x_i-k$), $|\mathcal{C}|$ clue residuals $x_i-g_i$, and $|\mathcal{P}|=810$ pairwise-difference residuals $x_i-x_j$ (each cell has $20$ peers, counted once). Equivalently, counting pairs per unit gives $27\binom{9}{2}=972$ terms, but $162$ of these pairs lie simultaneously in a row/column \emph{and} a box, leaving $810$ distinct pairs overall.

The first two sums are \emph{positive} regularisation that encourage $x_i\in D$ and enforce the clues. The final sum is \emph{negative} regularisation that rewards inequality: for digit-valued assignments it contributes $-\beta$ for every pair of unequal entries and $0$ for equal entries. The $810$ pairwise terms are essentially irreducible if one insists on \emph{pairwise} penalties: omitting a single peer-pair $(i,j)$ removes the only residual that directly distinguishes $x_i=x_j$ from $x_i\neq x_j$ once $x$ is digit-valued, and no linear ``hash'' of the remaining differences can substitute because nonvanishing is not preserved under linear combinations (e.g.\ $a-b\neq 0$ and $b-c\neq 0$ does not prevent $a-c=0$).

The compact objective in Proposition~\ref{prop:all-different} would use one domain penalty \(R_i\) per cell, with singleton domains on clues. Equation \eqref{eq:objective} is a more redundant but regression-friendly expansion of the same idea: it keeps the $729$ digit-snapping residuals visible and writes the clue penalties separately rather than folding them into the domains. This is not the tersest way to state the theorem, but it is the most convenient form for implementation and for counting the synthetic observations explicitly.

\begin{corollary}[Sudoku completion as a special case]
\label{cor:sudoku-special-case}
Let \(G_{\mathrm{Sud}}\) be the graph on the \(81\) cells of a Sudoku puzzle, with an edge between two cells exactly when they share a row, column, or \(3\times 3\) box. For a puzzle with clue set \(\mathcal{C}\) and clue digits \(g_i\), define
\[
D_i :=
\begin{cases}
\{g_i\}, & i\in \mathcal{C},\\
\{1,\dots,9\}, & i\notin \mathcal{C}.
\end{cases}
\]
Then the global minimisers of the objective from Proposition~\ref{prop:all-different} are exactly the valid completions of the Sudoku puzzle.
\end{corollary}

\begin{proof}
Each row, column, and box of Sudoku forms a clique of size \(9\) in \(G_{\mathrm{Sud}}\). Thus a proper list-colouring assigns pairwise distinct digits in each unit. Since the available values are exactly \(\{1,\dots,9\}\), pairwise distinctness in a unit is equivalent to containing every digit exactly once. The singleton domains enforce the clues. Apply Proposition~\ref{prop:all-different}.
\end{proof}

For Sudoku, this bookkeeping is completely explicit. In the current draft, \(\mathcal P\) is the set of \emph{distinct} peer pairs, so each cell has exactly \(20\) incident negative terms. Thus Proposition~\ref{prop:all-different} gives the uniform sufficient condition \(\lambda>20\beta\) in the deduplicated peer graph. If instead one rewrote the loss as a sum over row, column, and box scopes and counted overlaps separately, each cell would contribute \(8+8+8=24\) incident terms, and the corresponding sufficient constant would be \(24\beta\). This is the precise place where the negative regularisation has to be dominated by the positive unary weight.

\paragraph{Why $p$-adics help with negative weights.}
If one tries to do this with an ordinary squared-loss term $-(x_i-x_j)^2$ in $\mathbb{R}$, the optimiser is perversely incentivised to make some differences \emph{large}, not merely nonzero, and the objective can become unstable unless heavily bounded. With $p>9$ and digit-valued variables, $|x_i-x_j|_p\in\{0,1\}$, removing magnitude bias.

\section{A stepwise/coordinate-descent heuristic}

The objective \eqref{eq:objective} induces an NP-hard global optimisation problem (consistent with the author's NP-hardness result for $p$-adic regression). The goal here is not a polynomial-time solver, but a proof-of-concept heuristic.

In implementation we take the ``$\alpha\to\infty$'' limit by restricting search to digit-valued assignments consistent with clues. Concretely, we initialise each row as a random permutation of $1,\dots,9$ consistent with its clues, and then apply row-wise swaps that reduce the remaining loss (column/box conflicts). This is a form of coordinate descent on an integer lattice.

\begin{algorithm}[t]
\caption{Row-swap stepwise regression (heuristic)}
\begin{algorithmic}[1]
\Require Puzzle with clues $\mathcal{C}$, max steps $T$, restarts $R$
\For{$r=1$ to $R$}
    \State Initialise each row as a permutation of $1,\dots,9$ consistent with its clues
    \For{$t=1$ to $T$}
        \If{no column/box conflicts} \State \Return solution \EndIf
        \State Choose a row involved in a conflict
        \State Find the best swap of two non-clue cells in that row (largest loss decrease)
        \State Apply the swap (or a random swap if no improving swap exists)
    \EndFor
\EndFor
\State \Return best assignment found
\end{algorithmic}
\end{algorithm}

This ``stepwise regression'' view matches the informal idea: at each step, choose the coefficient change that gives the largest incremental loss improvement.

\section{A Zubarev-style stochastic walk}
The ``stepwise regression'' heuristic above is tailored to the strong-snapping Sudoku setting: evaluate all within-row swaps and greedily choose the best improving move (or a random move when stuck). Once we take seriously the idea that Sudoku is being treated as a regression instance, it is natural---if delightfully over the top---to also try a generic training method from the $p$-adic regression literature.

\subsection{Zubarev's proposal in the regression setting}
In \cite{zubarev2025padic}, Zubarev notes that loss functions built from sums of $p$-adic norms are locally constant almost everywhere, so gradient methods are uninformative. He therefore suggests a stochastic alternative: ``we can propose a stochastic optimization algorithm for the function (9) based on a random walk in the parameter space $w$''~\cite{zubarev2025padic}.

Concretely, the proposed training dynamics is a discrete-time random process
\begin{equation}
w_{t+1} = w_t + \xi_t(w_t,\beta_t),
\label{eq:zubarev-walk}
\end{equation}
where $\beta_t>0$ is a ``continuity'' (inverse temperature) parameter and $\xi_t$ is sampled from a distribution biased toward loss-decreasing moves. In Zubarev's formulation the conditional law has density (with respect to the $p$-adic Haar measure)
\begin{equation}
P(\xi \mid w,\beta)\ \propto\ \exp\!\bigl(-\beta\,(L(w+\xi)-L(w))\bigr).
\label{eq:zubarev-boltzmann}
\end{equation}
Under mild conditions (in particular, that the set of improving moves has nonzero measure), Zubarev shows there is a regime of $\beta$ values for which the expected loss decreases, i.e.\ $L(w_t)$ is a strict supermartingale~\cite{zubarev2025padic}.

\subsection{A discrete Sudoku analogue}
In our ``$\alpha\to\infty$'' implementation, the state is already digit-valued and clue-consistent. We therefore treat the Zubarev walk as a \emph{finite-move} stochastic optimiser:
\begin{itemize}
    \item The parameter vector $w$ is the current Sudoku filling $x\in D^{81}$.
    \item The loss $L(x)$ is the number of conflicting pairs in columns and boxes (equivalently, \eqref{eq:objective} up to constants in the digit-restricted regime).
    \item A move $\xi$ is a swap of two non-clue cells within a single row (preserving each row as a permutation of $1,\dots,9$).
\end{itemize}

For a chosen row $r$ and current state $x$, let $\Xi_r(x)$ be the set of all such swaps, and let $\Delta L(\xi) := L(x\oplus\xi)-L(x)$ be the change in loss after applying swap $\xi$ (where $x\oplus\xi$ denotes ``apply the swap''). Replacing the Haar measure in \eqref{eq:zubarev-boltzmann} with the counting measure on the finite set $\Xi_r(x)$ gives the discrete sampling rule
\begin{equation}
\Pr(\xi \mid x,\beta_t) = \frac{\exp\!\bigl(-\beta_t\,\Delta L(\xi)\bigr)}{\sum_{\xi'\in\Xi_r(x)} \exp\!\bigl(-\beta_t\,\Delta L(\xi')\bigr)}.
\label{eq:zubarev-discrete}
\end{equation}
When $\beta_t\to 0$ this is uniform random swapping; when $\beta_t\to\infty$ it concentrates on the best (most loss-decreasing) swap, recovering a soft version of the greedy stepwise algorithm. In practice we use a simple annealing schedule: $\beta_t$ increases linearly from $\beta_0$ to $\beta_1$ over the allotted steps, gradually shifting from exploration to exploitation.

\begin{algorithm}[t]
\caption{Zubarev walk over row swaps (heuristic)}
\begin{algorithmic}[1]
\Require Puzzle with clues $\mathcal{C}$, max steps $T$, restarts $R$, schedule $\{\beta_t\}_{t=1}^T$
\For{$r=1$ to $R$}
    \State Initialise each row as a permutation of $1,\dots,9$ consistent with its clues
    \For{$t=1$ to $T$}
        \If{no column/box conflicts} \State \Return solution \EndIf
        \State Choose a row involved in a conflict
        \State Enumerate row swaps $\Xi_r(x)$ and compute $\Delta L(\xi)$ for each $\xi\in\Xi_r(x)$
        \State Sample $\xi\in\Xi_r(x)$ with probability $\propto \exp(-\beta_t\,\Delta L(\xi))$
        \State Apply swap $\xi$
    \EndFor
\EndFor
\State \Return best assignment found
\end{algorithmic}
\end{algorithm}

\section{Compared with SAT/CSP encodings}

A standard SAT encoding introduces Boolean variables
\[
y_{r,c,d}\in\{0,1\},
\]
with the intended meaning that cell $(r,c)$ contains digit $d$. This gives
\[
9\times 9\times 9 = 729
\]
variables before any clues are applied.

In a direct CNF model, four families of exactly-one constraints are required:
\begin{enumerate}
    \item one digit per cell ($81$ constraints);
    \item one occurrence of each digit per row ($81$ constraints);
    \item one occurrence of each digit per column ($81$ constraints);
    \item one occurrence of each digit per box ($81$ constraints).
\end{enumerate}
Flattening each exactly-one constraint on $9$ literals into CNF gives one $9$-literal clause plus $\binom{9}{2}=36$ binary exclusion clauses. Hence the expanded SAT input contains
\[
324 \text{ long clauses} \;+\; 324\cdot 36 = 11{,}664 \text{ binary clauses},
\]
for a total of
\[
11{,}988 + |\mathcal{C}|
\]
clauses after adding one unit clause for each clue in $\mathcal{C}$.

The exact-cover / CSP view uses the same $729$ indicator variables, but packages them into $324$ structured constraints rather than flattening them all the way to CNF. By contrast, the $p$-adic regression model works with only $81$ digit-valued variables and the objective
\[
729 \text{ digit-snapping residuals} \;+\; |\mathcal{C}| \text{ clue residuals} \;+\; 810 \text{ pairwise residuals},
\]
so the fully expanded regression instance has
\[
1{,}539 + |\mathcal{C}|
\]
scalar terms. Relative to naive CNF, this is a factor of about
\[
\frac{11{,}988}{1{,}539} \approx 7.8
\]
fewer expanded constraints, together with a factor of
\[
\frac{729}{81} = 9
\]
fewer variables. Relative to exact cover, it still uses $9\times$ fewer variables, but more scalar terms than the $324$ structured constraints.

One simple information-theoretic proxy for model complexity is the entropy of a uniformly random unconstrained assignment. For the Boolean SAT variables this is
\[
H_{\mathrm{SAT}} = \log_2(2^{729}) = 729 \text{ bits},
\]
whereas for the $p$-adic Sudoku variables restricted to $D=\{1,\dots,9\}$ it is
\[
H_{p\text{-adic}} = \log_2(9^{81}) = 81\log_2 9 \approx 256.8 \text{ bits}.
\]
This is not a theorem about solver runtime, but it does quantify the sense in which the regression model is a compressed parameterisation: it exposes fewer free degrees of freedom. The row-wise permutation heuristic compresses further to $(9!)^9$, with entropy
\[
H_{\mathrm{row\;perm}} = 9\log_2(9!) \approx 166.2 \text{ bits}
\]
before clues, or more generally
\[
\sum_{r=1}^9 \log_2((9-c_r)!)
\]
if row $r$ contains $c_r$ fixed clues. The price of this compression is that the residuals are arithmetic and global rather than local and Boolean.

\section{Discussion and extensions}
The model is intentionally ``the hard way'': it embeds Sudoku into a (potentially) NP-hard regression-like optimisation problem rather than exploiting Sudoku structure directly.

\paragraph{Training Sudoku as a regression model.}
The most absurd aspect of the construction is arguably not the loss itself, but the fact that once we write down \eqref{eq:objective}, we can treat Sudoku completion as a \emph{training problem} and import generic training dynamics from the $p$-adic regression setting. The Zubarev walk is a particularly clean example: a loss-biased random walk that is not designed for Sudoku in any way, yet still finds valid completions when run over the row-swap move set.

\subsection{An alternative sparse encoding: powers-of-two sum constraints (archived)}
The objective \eqref{eq:objective} uses a large collection of pairwise inequality terms: $|\mathcal{P}|=810$ distinct peer pairs (equivalently $27\binom{9}{2}=972$ pairs if counted per unit, with overlap). An appealing alternative is to enforce all-different constraints \emph{without} pairwise penalties, by changing the digit representation.

Map digits $d\in\{1,\dots,9\}$ to powers of two,
\[
d \longmapsto 2^{d-1} \in A := \{1,2,4,\dots,256\}.
\]
If every cell variable is restricted to $A$, then the condition ``a unit contains each digit exactly once'' is equivalent to a \emph{single} sum constraint:
\[
\sum_{(r,c)\in g} \beta_{r,c} = 1+2+\cdots+256 = 2^9-1 = 511,
\]
for each row/column/box $g$. This collapses the Sudoku constraints to $27$ residuals of the form
\[
r_g(\beta) := 511 - \sum_{(r,c)\in g}\beta_{r,c}.
\]
For a prime $p$, one can then study the extremely parameter-sparse $p$-adic objective
\[
L_p(\beta) := \sum_g |r_g(\beta)|_p,
\]
optionally together with a (soft or hard) regulariser that encourages $\beta_{r,c}\in A$ and enforces the clues.

This encoding also suggests a ``Hensel-ish'' lifting perspective: a constraint is satisfied at lift level $k$ when $r_g(\beta)\equiv 0\pmod{p^k}$ (equivalently $v_p(r_g(\beta))\ge k$), and a heuristic tries to increase the minimum valuation across groups. Once $p^k > 9\cdot 256 = 2304$, having all residuals divisible by $p^k$ forces every residual to be zero, yielding an actual integer solution. In archived experiments with simple local-search moves (greedy single-cell changes and simulated annealing), this objective exhibited abundant local minima and did not reliably solve standard benchmark puzzles; however, the representation remains conceptually attractive because it uses so few structural constraints.

Several extensions are immediate:
\begin{itemize}
    \item \textbf{Confusion-aware digit losses.} If clues came from OCR or handwriting rather than clean symbolic input, one could weaken the loss for commonly confused pairs (for example $1$ versus $7$) instead of treating every wrong digit as equally wrong.
    \item \textbf{Alternative constraint encodings.} Beyond the powers-of-two sum encoding above, one can replace the inequality reward with difference regularisation over the finite set $\{\pm 1,\dots,\pm 8\}$, at the cost of a more complicated objective landscape.
\end{itemize}

\section{Conclusion}
This note takes a deliberately contrarian route to a familiar puzzle: it first shows that finite-domain all-different systems admit tidy $p$-adic regression encodings, and then studies Sudoku as a vivid special case. The construction is mathematically tidy precisely because the $p$-adic norm turns ``different from'' into an indicator-like reward once the variables are digit-valued.

As an optimisation method, this is obviously not competitive with mature Sudoku solvers or SAT/CSP technology. But the construction still earns its keep twice over: it gives a genuinely small model---$81$ digit-valued variables rather than $729$ Boolean indicators---and it shows that the same $p$-adic machinery used in earlier regression work is expressive enough to encode crisp all-different constraint systems in general. Even generic training dynamics such as the Zubarev walk can then recover valid solutions in a concrete case. In that sense the paper is less about Sudoku alone than about the surprising breadth of problems that can be phrased as ultrametric regression landscapes.
